Записи об играх на этом сайте публикует ИИ-агент (Claude Code), у которого есть доступ к репозиторию. Раньше я собирал данные для записи руками, теперь агент делает всё сам: читает код игры, разбирается в механике и создаёт готовую запись. Мне остаётся только положить картинку-превью.

## Промпт

```text
Опубликуй на сайте запись о новой игре из раздела «Игры».

Игра лежит в dev/public/lab/{имя-игры}/ — изучи её самостоятельно:
прочитай index.html и js-файлы, разберись, что это за игра, какая
у неё механика, как в неё играть и что в ней получилось
интересного (включая забавные баги, если они видны из кода).

Модель, которая сгенерировала игру: {модель}
Время разработки: {сколько заняло}

Создай запись dev/src/content/posts/{имя-игры}.md:

1. title — короткое название игры по-русски, без кавычек и слова «игра».
2. description — 1–2 предложения, не больше 160 символов: сначала
   суть игры, потом — чем она цепляет. Тон лёгкий и честный,
   самоирония уместна, канцелярит — нет. Запрещены слова
   «уникальный», «инновационный», «погрузиться» и восклицательные
   знаки. Это эксперимент, а не коммерческий релиз, — не продавай.
3. Фронтматтер: date — сегодня, category: games, model и timeSpent —
   из данных выше, gameUrl: /lab/{имя-игры}/,
   cover: /lab/{имя-игры}/cover.png (картинку я добавлю сам).
4. Текст записи — 2–4 абзаца: как играть, что вышло хорошо,
   что криво и почему. Пиши по делу, без воды.

Запись создай черновиком (draft: true), чтобы я посмотрел её
в npm run dev. Не собирай и не публикуй сайт — это я сделаю сам,
когда добавлю обложку и сниму черновик.
```

## Как пользоваться

1. Подставь имя папки с игрой, модель и время разработки.
2. Агент прочитает код игры и создаст черновик записи.
3. Положи скриншот игры в `dev/public/lab/{имя-игры}/cover.png` — это единственное, что делается руками.
4. Проверь черновик в `npm run dev`, убери `draft: true` — и публикуй.

Ограничение в 160 символов для description не каприз: более длинные описания обрезаются в поисковой выдаче и расползаются в карточках.
